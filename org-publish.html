1 2019-05-24 开始使用

今天重新捡起了静态发布.

部署一个Wordpress并不费劲, 但是我想专注于内容. 已经习惯了Emacs中的编辑工作. 因为之前有在Hexo, jekyll, Hugo和org-publish中考察试用过. 当时放弃Hexo的原因主要在于Hexo对org-mode的支持:

  1. 之前版本的Emacs编辑md文件, 插入代码的编辑会报错, 而我并不想解决这些错误;
  2. 换用Markdown会得到较好的支持, 但md 文件不能方便的运行片段并获得结果;

目前对生成进行了自定义, 取消了自带样式, 引入了bootstrap, 并定义了一个简单的模板;

3 遗留问题

  1. [ ] 锚点链接不正确可能导致发布失败;
  2. [ ] 因为要检查文件是否被修改, 在数量较多(超过200个)时会导致发布变慢;
  3. [ ] 资源文件如css, js的修改也需要发布, 稍微麻烦了一些;
  4. [ ] 块元素如果是代码块, 发布时会询问是否需要执行, 无法自动化;
